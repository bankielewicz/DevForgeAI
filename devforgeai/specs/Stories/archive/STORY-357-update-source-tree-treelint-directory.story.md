# Story: Update source-tree.md with .treelint/ Directory Structure

## Description

**As a** DevForgeAI AI agent (subagent or skill reading context files),
**I want** the `.treelint/` directory structure documented in source-tree.md at the project root level,
**so that** I know where Treelint artifacts (index database, configuration, and daemon socket) belong in the repository structure and can correctly reference these locations during code search, validation, and project setup workflows.

## Notes

**Design Decisions:**
- Placement: `.treelint/` placed after `.claude/` to keep dot-prefixed directories grouped together at top of tree diagram
- Distinguished from `src/bin/treelint/` (distribution binaries) vs `.treelint/` (runtime working directory)
- Glob visibility note: `.treelint/` starts with dot, so Claude Code Glob tool may skip it (same as `.claude/`)

**Edge Cases Documented:**
- Windows compatibility: daemon.sock is Unix-only; Windows may use named pipe or TCP fallback
- Large index.db: Can grow to hundreds of MB for large codebases; must be gitignored
- Tree diagram formatting: Must use correct Unicode box-drawing characters (├──, └──, │)
