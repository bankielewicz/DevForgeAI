```

  prompt="Generate comprehensive test suite for this story.

  Story content is already loaded in conversation (via @file reference from /dev command).

  1. Acceptance criteria (Given/When/Then scenarios)
  2. Technical specification (API contracts, data models, business rules)
  3. Non-functional requirements (performance, security)

  Context files available:
  - devforgeai/specs/context/source-tree.md (test file placement rules)
  - devforgeai/specs/context/coding-standards.md (test patterns, AAA format)
  - devforgeai/specs/context/tech-stack.md (test framework: {TEST_FRAMEWORK})

  **source-tree.md Context for test file placement:**
  - Module Under Test: ${MODULE_PATH}
  - Expected Test Directory: ${TEST_DIRECTORY} (per source-tree.md)
  - Constraint: All generated tests must be placed in ${TEST_DIRECTORY}

  Generate tests that:
  1. Cover all acceptance criteria
  2. Follow AAA pattern (Arrange, Act, Assert)
  3. Use test framework: {TEST_FRAMEWORK}
  4. Place tests according to source-tree.md rules in ${TEST_DIRECTORY}
  5. Initially FAIL (Red phase of TDD)

  Test command to verify: {TEST_COMMAND}

  Return:
  - Test files created (paths and content summary)
  - Test count (unit/integration/e2e)
  - Initial test run status (all should fail)

  **Treelint Integration:** This subagent is Treelint-enabled. For AST-aware code search on supported languages, use Treelint patterns from `.claude/agents/references/treelint-search-patterns.md`. Fallback to Grep when unavailable."

```

### Step 4: Technical Specification Coverage Validation & Deferral Pre-Approval [MANDATORY]

**Purpose:** Ensure ALL Technical Specification components have tests. Prevents autonomous deferrals where stubs pass tests but skip real implementation.

**Execution:** After Steps 1-3, before Phase 03.

#### 4.1 Extract Technical Specification Components

Story file already loaded via @file reference from /dev command.

**RCA-006: Detect format version from frontmatter:**
```python

```

**If format_version == "2.0" (Structured YAML):**
```python

    component_checklist.append({
```

Build component checklist with type, name, file path, and requirements array.

#### 4.4 Request User Decision for EACH Gap

```python

        "options": [

            {"label": "Defer to follow-up story", "description": "Skip now, create tracking story. Creates technical debt."},

        ]
```

---

#### 4.5 Process User Decision

**Generate tests now:** Re-invoke test-automator with specific missing requirements. Add to Phase 02 test suite.

**Defer to follow-up story:**
- Ask user: create new story or use existing story ID
- If new: invoke `Skill(command="devforgeai-story-creation")`
- Document deferral record: component, requirements, reason, follow-up story, user approval timestamp
- Add to Phase 06 tracking

**Remove from scope:**
- Require ADR documenting why requirements removed
- HALT Phase 02 until ADR created and story file updated
- Re-run /dev after updates complete

Process EACH component with gaps. Update coverage map with user decisions. Track all decisions in `DECISIONS_LOG` array.

### Reflection Capture on Failure (EPIC-051)

**When:** AFTER phase validation fails, BEFORE retry attempt.

**Step 1: Generate Reflection**
```json
{
  "id": "REF-02-{timestamp}",
  "phase": "02",
  "failed": true,
  "iteration": "{count_existing_phase_02_reflections + 1}",
  "reflection": {
    "what_happened": "[Specific test failure: test name, assertion error, line number]",
    "why_it_failed": "[Root cause: incorrect expected value, missing setup, wrong assertion]",
    "how_to_improve": "[Actionable fix: re-read AC#N, check test setup, verify assertion]"
  },
  "timestamp": "{ISO8601_format}"
}
```

**Step 2:** Read existing reflections from `devforgeai/workflows/${STORY_ID}-phase-state.json`, calculate iteration count.

**Step 3:** Append reflection to phase state reflections[] array.

**Step 4:** On retry (iteration > 1), display previous reflection's what/why/how before starting.

**ID uniqueness:** `REF-{phase}-{timestamp}` with millisecond precision ensures no collisions.

**Error handling:** Reflection capture failure logs warning but never blocks retry (NFR-002).
