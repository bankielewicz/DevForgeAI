Feature: Product Listing with Filtering and Pagination

Display products with comprehensive filtering, sorting, and pagination capabilities.

Acceptance Criteria:
1. List displays all products with name, price, description, SKU, and image URL
2. Pagination supports configurable page size (default 20, max 100 items per page)
3. Filtering by category, price range, availability status, and rating
4. Sorting options: price (asc/desc), popularity (view count), date added (newest first), rating
5. Search integration: filter combined with text search by product name and description
6. Catalog page loads in under 2 seconds for 1000+ products
7. Database query uses indexed columns for filter and sort operations

Non-Functional Requirements:
- API response time: <500ms for queries with filters
- Support 100+ concurrent users
- Cache product list in Redis with 1-hour TTL
- Database indices on category, price, and created_date columns
- Responsive design works on mobile, tablet, desktop

Technical Specification:
- Endpoint: GET /api/products?page=1&page_size=20&category=electronics&sort=price_asc
- Database: PostgreSQL with products table (id, name, price, category, created_at, view_count)
- Caching: Redis cache invalidated on product updates
- Performance: query optimization with EXPLAIN ANALYZE, index analysis
