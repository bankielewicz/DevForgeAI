Background job queue using RabbitMQ. Process 10,000 jobs per minute.

Job types: email batches (5 min), customer reports (30 min), data imports (15 min).

Given: End user starts bulk import with 10K rows.
When: Job added to queue.
Then: Instant confirm, import done in 600 seconds.

Given: Background job encounters error (DB down).
When: Error occurs during processing.
Then: Retry 3 times with backoff (5s, 10s, 30s).

Given: Job fatal error.
When: All retries exhausted.
Then: Move to dead letter queue for admin user review.

Status tracking: queued, running, done, failed.
End users track job by ID. Logs with timestamps and errors.

Performance requirement: 10,000 jobs per minute.
Reliability requirement: Retry logic and checkpoints.
