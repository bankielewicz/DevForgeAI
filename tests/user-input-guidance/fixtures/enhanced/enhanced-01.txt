Feature: User Authentication API

Create a REST API endpoint for user registration with comprehensive input validation and security measures.

Acceptance Criteria:
1. Users can register with email and password
2. Email format validation using RFC 5322 standard
3. Password strength requirements: minimum 12 characters, uppercase, lowercase, number, special character
4. Duplicate email detection prevents duplicate accounts
5. User data persists to PostgreSQL database
6. Success response includes user ID, email, and creation timestamp
7. Password is hashed using bcrypt with salt rounds ≥12

Non-Functional Requirements:
- Registration endpoint responds within 500ms under normal load
- Handles concurrent registrations without race conditions
- Implements rate limiting: max 10 registration attempts per IP per hour
- Logs all registration attempts with timestamp and IP address
- Database transactions ensure atomic operations

Technical Specification:
- Endpoint: POST /api/auth/register
- Database: PostgreSQL with user table (id, email, password_hash, created_at)
- Libraries: fastapi, pydantic for validation, bcrypt for hashing
- Error handling: return 409 for duplicate email, 400 for validation errors
