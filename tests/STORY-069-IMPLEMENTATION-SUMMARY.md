## Design Patterns Followed

### 1. **Dependency Injection**
All modules accept paths as parameters (no hardcoded paths):
```python
verify_bundle_integrity(bundle_root: Path) -> dict
install_python_cli_offline(bundle_root: Path, target_dir: Path) -> dict
```

### 2. **Single Responsibility Principle**
Each module has one clear purpose:
- `network.py` → Network detection only
- `checksum.py` → Integrity verification only
- `offline.py` → Offline installation workflow
- `bundle.py` → Bundle structure validation

### 3. **Graceful Degradation**
Optional features skip with warnings (don't halt installation):
- Python unavailable → Warn and continue
- Network unavailable → Use offline mode automatically
- Checksum warnings (≤2 failures) → Warn but continue
- Checksum errors (≥3 failures) → Halt (tamper detection)

### 4. **Type Hints and Docstrings**
All functions have:
- Type hints for parameters and return values
- Docstrings with Args, Returns, Raises, Examples
- Python 3.10+ style (using `|` for unions)

### 5. **Standard Library Only**
No external dependencies:
- `socket` for network detection
- `hashlib` for SHA256 checksums
- `subprocess` for Python CLI installation
- `pathlib` for file operations
- `json` for manifest parsing
