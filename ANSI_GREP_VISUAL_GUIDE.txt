YOUR PROBLEM
────────────────────────────────────────────────────────────────────────────────

Error: grep: Unmatched [, [^, [:, [., or [=

When running:
  $ grep -q "${GREEN}"
  where GREEN='\033[32m'


WHY THE ERROR OCCURS
────────────────────────────────────────────────────────────────────────────────

Pattern analysis:
  \033[32m
  ├─ \033 = Escape character (OK)
  ├─ [ = START of bracket expression (regex syntax)
  ├─ 32m = Contents of bracket expression
  └─ MISSING: Closing ] bracket (ERROR!)

Grep expects: [abc] or [a-z] or [^xyz]
             (balanced bracket expression)

You're providing: [32m (incomplete bracket expression)
                 └─ Unmatched opening bracket!


THE FIX (Solution 1 - RECOMMENDED)
────────────────────────────────────────────────────────────────────────────────

BEFORE (ERROR):
  if echo "${output}" | grep -q "${GREEN}"; then

AFTER (FIXED):
  if echo "${output}" | grep -qF "${GREEN}"; then
                               ↑
                         Add -F flag


WHAT -F DOES
────────────────────────────────────────────────────────────────────────────────

-F flag = "Fixed strings" = treat pattern as literal text, NOT regex

╔─────────────────────────────────────────────────────────────────────────────╗
│ Without -F: grep INTERPRETS pattern as regex                               │
│   [32m → "This is a bracket expression containing 3, 2, m"                 │
│   Result: ERROR (incomplete bracket)                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│ With -F: grep MATCHES pattern as literal string                            │
│   [32m → "Find the exact bytes: [, 3, 2, m"                                │
│   Result: SUCCESS (matches ANSI code exactly)                              │
└─────────────────────────────────────────────────────────────────────────────┘

UNDERSTANDING ANSI CODES
────────────────────────────────────────────────────────────────────────────────

ANSI Color Code Structure:
  ESC [ Parameters m
  │   │  │          │
  │   │  │          └─ 'm' terminates the sequence
  │   │  └────────────── Numeric parameter (color, style, etc.)
  │   └───────────────── Control Sequence Introducer
  └───────────────────── Escape character

Examples:
  \033[32m  = Green text
  \033[33m  = Yellow text
  \033[31m  = Red text
  \033[0m   = Reset to default

In bash:
  $(printf '\033[32m')  = Generate green ANSI code
  '\033[32m'            = Literal ANSI code (string)
