  Spec (YAML) → Tests (TDD Red Phase)  ✓ CORRECT - tests match spec
                       ↓
           Tests expect specific signatures
                       ↓
        Implementations written with DIFFERENT signatures ✗ WRONG

RESULT: Tests fail at instantiation stage (can't even create objects)
        No business logic executed, pure interface mismatch


WHY THIS HAPPENED:

Implementations were written WITHOUT referencing tests first.
- Backend architect interpreted YAML spec differently than test-automator
- Example: "backup location" → spec ambiguous
  - Test chose: target_dir + files_to_backup (explicit parameters)
  - Implementation chose: source_paths (combined parameter)

This is an anti-pattern in TDD: implementations should FOLLOW tests, not guess spec intent
