# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in {{project_name}}, **please do not open a public issue.**

Instead, report it via one of these channels:

- **GitHub Security Advisories:** [Report a vulnerability]({{repo_url}}/security/advisories/new) (preferred)
- **Email:** {{contact_email}}

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact assessment
- Suggested fix (if you have one)

### Response Timeline

| Action | Target |
|--------|--------|
| Acknowledge receipt | 48 hours |
| Initial assessment | 5 business days |
| Fix timeline communicated | 10 business days |
| Public disclosure | After fix is released |
